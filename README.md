# AI BIBLE

Bible study + evangelism tool. Goal: bring clearer understanding of Scripture and offer hope grounded in God's Word — eventually packaged for Google Play.

## Translations on disk

| Translation | License | Source | Books | Bundled? |
|---|---|---|---|---|
| **KJV** (King James Version, 1611) | Public domain | [thiagobodruk/bible](https://github.com/thiagobodruk/bible) | 66 | ✅ |
| **BSB** (Berean Standard Bible, 2022) | Public domain (CC0) | [bereanbible.com](https://bereanbible.com) | 66 | ✅ |
| **WEB** (World English Bible) | Public domain | API.Bible (`9879dbb7cfe39e4d-01`) | 66 | ✅ |
| **ASV** (American Standard Version, 1901) | Public domain | API.Bible (`06125adad2d5898a-01`) | 66 | ✅ |
| **NKJV** (New King James Version) | **Licensed — runtime only** | API.Bible (`63097d2a0a2f7db3-01`) | — | runtime fetch |
| **NIV** (New International Version) | **Not available on Starter plan** | Requires separate Biblica approval | — | — |

> **Licensed translations** (NKJV, NIV) are fetched per-verse at runtime through API.Bible and never written to disk in full. Shipping any licensed translation in a Play Store app requires the underlying license to permit it (Biblica for NIV, Thomas Nelson for NKJV — both restrictive).
>
> **NIV access:** API.Bible's Starter plan (5K calls/mo) does **not** include NIV. To add it, request it through your application's "Add Bibles" workflow on api.bible — Biblica reviews each application individually. NKJV is included by default and gives a similar modern-English reading experience.
>
> **KJV count note:** the canonical KJV verse count is 31,102. This digitization reports 31,100 — a known 2-verse discrepancy from edition differences (e.g., split/joined verses in 3 John, Revelation 13). Acceptable for study; flag if doing exact-citation work.

## Two front-ends

### CLI — Hope Finder

```powershell
node bin/bible.mjs help                      # usage
node bin/bible.mjs topics                    # list 35 curated topics
node bin/bible.mjs hope                      # find verses for "hope"
node bin/bible.mjs anxiety                   # alias → "fear" topic
node bin/bible.mjs ref "John 3:16"           # KJV + BSB lookup
node bin/bible.mjs compare "John 3:16"       # KJV + BSB + NKJV side-by-side
node bin/bible.mjs search "mustard seed"     # full-text BSB search
node bin/bible.mjs vod                       # verse of the day
node bin/bible.mjs ask "what about anxiety?" # Claude-powered Q&A (needs ANTHROPIC_API_KEY)
```

On Windows, `bible.cmd` is a shim — drop the project folder on PATH and just type `bible hope`.

NKJV verses fetched via API.Bible are cached to `data/nkjv/cache.json` — each unique verse uses one API call once, then forever from disk.

### Web app (PWA → Play Store)

```powershell
node scripts/build_web.mjs        # copy data into web/data/
node scripts/dev_server.mjs       # http://localhost:8080
```

Tabs: **Daily** (verse of the day), **Topics** (35 curated situations), **Lookup** (any verse, KJV+BSB), **Search** (full-text BSB).

PWA — installs to home screen, works fully offline after first load (service worker caches everything).

For Play Store packaging, see [PUBLISHING.md](PUBLISHING.md).

### JSON shape (both KJV and BSB)

```js
[
  {
    "book": "Genesis",          // BSB only — KJV uses "abbrev" instead
    "abbrev": "gn",             // KJV only
    "chapters": [
      [ "verse 1 text", "verse 2 text", ... ],   // chapter 1
      [ "verse 1 text", ... ],                   // chapter 2
      ...
    ]
  },
  ...
]
```

The two shapes are intentionally close but not identical — KJV preserves the upstream `abbrev` field; BSB uses full canonical book names. A unified-shape adapter belongs in app code, not in the raw data.

### Plain-text shape

```
# Genesis

## Genesis 1
1:1 In the beginning God created...
1:2 ...

## Genesis 2
2:1 ...
```

## Rebuilding

```powershell
node scripts/build_text_and_json.mjs
```

Reads `kjv.json` + `bsb_raw.txt`, regenerates `kjv.txt`, `bsb.json`, `bsb.txt`. Idempotent.

## API.Bible (working)

- **Endpoint:** `https://rest.api.bible/v1`
- **Auth header:** `api-key: <key>`
- **Plan:** Starter, 5,000 calls/month
- **Key:** stored in `.env` as `API_BIBLE_KEY`
- **Bible IDs we use:**
  - NKJV: `63097d2a0a2f7db3-01`
  - KJV: `de4e12af7f28f599-01` (already on disk; useful for cross-checks)

Example verse fetch:
```bash
curl -H "api-key: $API_BIBLE_KEY" \
  "https://rest.api.bible/v1/bibles/63097d2a0a2f7db3-01/verses/JHN.3.16?content-type=text"
```

Verse refs use OSIS-style IDs: `BOOK.CHAPTER.VERSE` (e.g., `GEN.1.1`, `PSA.23.1`, `REV.22.21`).

## Optional: enable `bible ask` (Claude Q&A)

The `ask` command uses Claude to answer questions from Scripture, grounded in our curated topic verses. Without a key, it gracefully tells you what to do.

```powershell
# 1. Get a key from https://console.anthropic.com
# 2. Add to .env:
ANTHROPIC_API_KEY=sk-ant-...
```

Example: `bible ask "is it okay to be angry?"` — pulls the **anger** topic's verses as grounding, asks Claude to weave a faithful answer that quotes from them.
